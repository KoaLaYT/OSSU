A little summary of the course -- How To Code: Simple Data

This is the final project of How To Code: Simple Data, but written in HTML and JS!

In this course, I learnt how to split a complex problem such as this game into several parts, so that at a time, I only have to focus one simple things, but still get the whole programm architecture in mind.

This all happened because of three excellent recipes:

  • How to Design Data: is the fundation, the data structure will strongly affect the function design and overall world design.
  • How to Design Function: the template is very useful especially in self referenced data structure. I think I understand recursive much better after this courses.
  • How to Design World: this make me write more clean and structured code, which is way more readable and maintanable.

As I am going to be a formal frontend programmer in this September, I decided to practice these skills in my day to day used language: JS!

Here is how I wrote this game:

  1. write the big-bang framework in JS:

    I only implement the on-tick, to-draw, stop-when and on-key functionalities for the simplicity. With the help of window.requestAnimationFrame, the work is fairly easy.

  2. then, just follow the recipe of HtDW:
    1. domain analysis: The is game is fairly simple, and all the analysis is already made by the course teacher.
    2. define constants
    3. define data definitions
    4. write the structure of whole program, and inplement every helper functions.
